HTML Forms

Introduction to HTML Forms

HTML forms are used to collect user input and send it to a server for processing. Forms are essential for user interaction on websites, enabling features like:

  • User registration and login
  • Contact forms and feedback
  • Search functionality
  • Online surveys and polls
  • E-commerce checkout processes
  • File uploads

Basic Form Structure

Every HTML form starts with the <form> element:

<form action="submit-url" method="POST">
    <!-- Form elements go here -->
</form>

Form Attributes:

action
Specifies where to send form data when submitted
method
Defines how to send form data (GET or POST)
enctype
Specifies how form data should be encoded (important for file uploads)
target
Specifies where to display the response (same window, new window, etc.)
autocomplete
Controls whether browsers should autocomplete form fields
novalidate
Disables browser validation

Form Best Practices Summary

✅ Do:

  • Always use proper <label> elements
  • Provide clear instructions and help text
  • Use appropriate input types for better UX
  • Group related fields with <fieldset>
  • Implement proper validation (both client and server-side)
  • Make forms keyboard accessible
  • Provide clear error messages
  • Use HTTPS for sensitive data
  • Test with screen readers

❌ Don't:

  • Use placeholder text as labels
  • Make forms unnecessarily long
  • Forget to validate on the server side
  • Use generic error messages
  • Ignore mobile users
  • Auto-submit forms without user consent
  • Collect unnecessary personal information
